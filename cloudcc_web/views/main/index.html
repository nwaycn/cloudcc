简要说明书

使用前说明

1. 此平台支持各版本的FreeSWITCH 2. FreeSWITCH的路径建议为/usr/local/freeswitch
配置路径需为/usr/local/freeswitch/conf 3. 如果FreeSWITCH的路径不为/usr/local/freeswitch
那么启动时需要指定路径 freeswitch -conf /usr/local/freeswitch/conf 4. 如果要记录通话记录则一定要使用mod_cdr_pg_csv
也就说是编译的时候一定要和Postgresql一起使用 5. 此应用在早期开发时，曾开源最早的源代码，但由于很多第三方企业或个人当时就申请了著作权
及商标等，故后续版本取消代码开源

更多能力增强

1.呼叫中心中间件 2.智能客服、智能外呼 3.视频客服、视频外呼、智能视频客服 4.指挥调度+对讲+调音台+视频 5.门禁系统SDK 6.WebRTC网关 7.呼叫系统实时质检及座席助手 8.外呼回铃检测 9.可靠性录音系统 10.微信及其它协议的小程序语音客服

使用要求

1.产品主体部分开发分为三层：Web层、RPC服务、注册服务 2.数据库使用Postgresql 9.x以及后续版本 3.需要导入一个总的cloudcc_web的库，以及每个节点有一个cloudcc的库 4.如果能看到此页面，代表web服务已启动，且数据库是正常的 5.Web页面是同各节点的nway_cmd_server通信，同时由cloudcc_server提供注册服务 6.Web的数据库配置是在/opt/cloudcc/cloudcc_web/conf/app.conf 7.注册服务的数据库配置是在/opt/cloudcc/cloudcc_server/cloudcc.conf 8.RPC服务的数据库配置是在/opt/cloudcc/nway_cmd_server/cmd_server.conf
